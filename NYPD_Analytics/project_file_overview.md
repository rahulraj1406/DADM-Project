# NYPD Complaint Data Analytics - File Overview

This document provides a concise summary of every file and folder generated by the analytics pipeline.

## Python Scripts (Pipeline Execution)
*   **`stage1.py`**: Loads the raw dataset, computes basic schema statistics, and outputs the `data_dictionary.csv`.
*   **`stage2.py`**: Performs Exploratory Data Analysis (EDA). Parses temporal data, flags geographic outliers, and generates plots inside `eda_report/`.
*   **`stage3.py`**: Cleans the dataset (drops duplicates, critical nulls, and logical invalidities). Imputes demographics with 'UNKNOWN'. Outputs `nypd_clean.csv`.
*   **`stage4.py`**: Creates spatial & temporal features, KMeans clusters, and one-hot/ordinal encodings. Outputs `nypd_transformed.csv`.
*   **`stage5.py`**: Conducts Hypothesis tests, analyzes correlations, and draws `maps/` using Folium.
*   **`stage6.py`**: Trains the machine learning models: Task A (Binary Felony Classification), Task B (Multi-class Offense Type), and Task C (Prophet Time-Series forecasting).
*   **`stage7.py`**: Analyzes model interpretations. Evaluates SHAP feature impact, plots Prophet validation trajectories, and drafts `policy_recommendations.txt`.
*   **`stage8.py`**: Compiles charts into presentable PNG slides in `presentation/` and designs `dashboard_spec.md`.
*   **`stage9.py`**: Automates documentation generation (`final_report.pdf`, `README.md`, `requirements.txt`) and sets up git version-control tags.
*   **`stage10.py`**: Analyzes machine learning False Positive rate disparities and exports `fairness_audit_report.md`.

## Data & Dictionaries
*   **`data_dictionary.csv`**: High-level mapping of the original dataset columns, showcasing null counts and data types.
*   **`nypd_clean.csv`**: The dataset directly following Stage 3 missingness and outlier handling.
*   **`nypd_transformed.csv`**: The dataset following Stage 4 scaling and ML encoding ready for training.
*   **`encoder_mappings.json` & `scaler.pkl`**: Rules utilized to transform the raw categorical inputs into model-ready integers/floats.

## Models & Metrics
*   **`model_taskA.pkl`**: The exported optimal binary classification model (predicting IS_FELONY).
*   **`model_taskB.pkl`**: The exported optimal multi-class model (predicting OFFENSE_CATEGORY).
*   **`model_taskC_precinct_*.pkl`**: Prophet time models individually fitted to the 5 busiest police precincts.
*   **`model_results.csv`**: Side-by-side performance breakdown for every algorithm (F1, Accuracy, Precision, MAPE).
*   **`hypothesis_tests.csv`**: Output parameters of the Kruskal-Wallis & Chi-Square tests run during Stage 5.

## Data Visualizations
*   **`eda_report/`** (Folder): General exploratory histograms, pie-charts, and density graphs.
*   **`maps/`** (Folder): `kde_hotspots.html` mapping the geographical density of complaints.
*   **`confusion_matrices/`** (Folder): Classification heatmaps indicating model True vs False Positive distributions.
*   **`presentation/`** (Folder): Auto-generated 16:9 aspect ratio PNG slides for briefing stakeholders.
*   **`shap_*.png` & `feature_importance_chart.png`**: Plots evaluating precisely which features most deeply impact predicting severe crimes.

## Documentation & Text Logs
*   **`pipeline_log.txt`**: A literal step-by-step trace of every data manipulation or model building action the pipeline underwent.
*   **`future_work_roadmap.md`**: Guide for incorporating 311 service data, weather metrics, and deploying deep learning architecture.
*   **`fairness_audit_report.md`**: Breakdown determining whether models mistakenly target certain racial demographics at inequitable rates.
*   **`policy_recommendations.txt`**: Five specific, actionable strategies for the NYPD drawn mathematically from model outcomes.
*   **`README.md`**: Main repository guide detailing setup requirements and execution flow.
*   **`final_report.pdf`**: Formatted summary text wrapping the entire experiment for a non-technical audience.
